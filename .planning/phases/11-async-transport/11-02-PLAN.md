<objective>
Convert MCP tool handlers to async, create async singleton client management, and add sync-over-async wrappers in the JaegerClient facade.

Purpose: ASYNC-04 — The MCP tools must be async to call the httpx.AsyncClient directly (avoiding thread pool overhead). The JaegerClient facade must keep its sync public API so existing consumers don't need async/await. This plan bridges the async transport (Plan A) with the sync facade contract.

Output: `_mcp.py` with async get_client(), `tools.py` with 7 async handlers calling `await client.aget()`, `facade.py` with sync methods wrapping async internals via `asyncio.run()`.
</objective>

**_mcp.py — convert to async singleton:**

1. Replace `import threading` with `import asyncio`.
2. Replace `_client_lock = threading.Lock()` with `_client_lock = asyncio.Lock()`.
3. Convert `get_client` to `async def get_client() -> JaegerHTTPClient`:
   ```python
   _client: JaegerHTTPClient | None = None
   _client_lock = asyncio.Lock()
   
   async def get_client() -> JaegerHTTPClient:
       global _client
       if _client is None:
           async with _client_lock:
               if _client is None:  # double-checked locking
                   _client = JaegerHTTPClient()
       return _client
   ```

4. Update `app_lifespan` to use async lock and async close:
   ```python
   @asynccontextmanager
   async def app_lifespan(_app: FastMCP) -> AsyncIterator[dict[str, Any]]:
       logger.debug("jaeger_mcp: startup")
       try:
           yield {}
       finally:
           global _client
           async with _client_lock:
               if _client is not None:
                   try:
                       await _client.aclose()
                   except Exception:
                       logger.warning("jaeger_mcp: error closing HTTP client on shutdown", exc_info=True)
                   _client = None
           logger.debug("jaeger_mcp: shutdown — HTTP session closed")
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sync facade → async internals | asyncio.run() boundary — sync callers get blocking behavior |
| MCP tools → event loop | async tools run in FastMCP's event loop directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-05 | D (Denial of Service) | asyncio.run() in facade nested in event loop | mitigate | asyncio.run() creates a new event loop per call. If facade is called from within an existing event loop (unlikely for sync callers), it will raise RuntimeError — this is correct behavior. Document: facade is for sync contexts only. |
| T-11-06 | T (Tampering) | async singleton race | mitigate | asyncio.Lock with double-checked locking in get_client() prevents duplicate client creation. |
| T-11-07 | I (Info Disclosure) | credential exposure in async context | mitigate | aclose() zeros credentials same as sync close(). lifespan shutdown always calls aclose(). |
</threat_model>
